1. What is an App?
An App (Application) is a software program designed to perform a specific task for users.
Examples:
Writing documents
Sending messages
Playing music
Browsing the internet
Examples of apps:
Microsoft Word → create documents
Google Chrome → browse internet
WhatsApp → send messages
In simple words:
App = software built to help users do something.

2. What is a Mobile App?
A Mobile App is an application designed specifically to run on smartphones or tablets.
Mobile apps are built for mobile operating systems like:
Android
iOS
Examples of mobile apps:
Instagram

Google Maps
These apps are installed from app stores such as:
Google Play Store
Apple App Store

To understand Android development properly, you must know how a mobile app actually works internally. Think of a mobile app as four main layers working together:

User → UI → Logic → Operating System → Device Hardware
Let’s break it down step-by-step.

1. The Basic Flow of a Mobile App
When you open an Android app (for example Instagram), the following happens internally:
User taps the app icon
The Android launches the app
The first Activity (screen) opens
UI elements appear (buttons, text, images)
User interacts with UI
Logic code runs
OS communicates with hardware (camera, internet, storage)

2. Activity (The Screen of the App)
In Android, every screen is called an Activity.
Example:
If you open WhatsApp
You see different screens:
Chat list screen
Chat conversation screen
Settings screen
Each of these is a separate Activity.
Example structure:
MainActivity → Home screen
ChatActivity → Chat screen
ProfileActivity → User profile
The first activity that opens when an app starts is called the Main Activity.

3. UI (User Interface)
The UI is everything the user sees and interacts with.
Examples of UI components:
Buttons
Text fields
Images
Lists
Menus
These are defined using XML layout files in Android Studio.
Example UI elements:
TextView → shows text
Button → user clicks
ImageView → shows image
RecyclerView → list of items
Example:
When you open YouTube
UI contains:
Video thumbnails
Play button
Search bar

4. Logic (The Brain of the App)
Logic is the code that controls what happens when users interact with the UI.
This is written in:
Kotlin
Example:
User clicks Login button
Logic:
1. Read username
2. Read password
3. Send request to server
4. Check response
5. Open Home Screen
Example Kotlin idea:
button.setOnClickListener {
    loginUser()
}
So:
UI shows things → Logic decides what happens

5. Android Operating System Interaction
Your app does not directly control the phone hardware.
Instead it communicates with:
Android
The OS acts like a manager.
Example requests:
App asks Android OS:
"Open camera"
"Get GPS location"
"Access internet"
"Store data"
Android checks permissions and then allows access.
Example:
When you use Google Maps
Flow:
App → Android OS → GPS hardware → Location returned

6. Interaction with Phone Hardware

Introduction to Kotlin

What is Kotlin?
Kotlin is a modern programming language used to build applications such as:
Android apps 
Web applications 
Backend systems 
It is officially supported by Android, which means most Android apps today are written in Kotlin.

Why Kotlin was Created?
Before Kotlin, Android developers mainly used Java.
But Java had some problems:
Too much boilerplate (long code)
Null pointer crashes
Less modern features

Kotlin was created to:
Write less code
Be safer
Be more readable

Key Features of Kotlin
Simple and Concise Kotlin reduces code compared to Java.
Example:
fun main() {
    println("Hello Kotlin")
}
Interoperable with Java
Kotlin works perfectly with Java.
Meaning:
You can use Java code in Kotlin
You can use Kotlin in existing Java projects
Null Safety (Very Important)
Kotlin helps prevent crashes caused by null values.
This is one of its biggest advantages.
Object-Oriented + Functional
Kotlin supports:
OOP (Classes, Objects)
Functional programming (lambdas, higher-order functions)
Official Language for Android
Google announced Kotlin as the official Android language.
So most modern apps like:
Pinterest
Netflix
use Kotlin.

Basic Structure of a Kotlin Program
Every Kotlin program has a main function (entry point).
fun main() {
    println("Hello World")
}
Breakdown:
fun → keyword to define a function
main() → starting point
println() → prints output

Variables in Kotlin
Kotlin has two types of variables:
val (Immutable)
val name = "Vineesha"
Cannot be changed later
var (Mutable)
var age = 21
age = 22
Can be changed

Data Types in Kotlin
Kotlin automatically understands types.
val num = 10        // Int
val price = 10.5    // Double
val name = "Hi"     // String
val isTrue = true   // Boolean
This is called Type Inference

Output in Kotlin
println("Hello")
print("World")
Difference:
println() → next line
print() → same line

String Templates (Important)
Kotlin allows easy string formatting.
val name = "Vineesha"
println("Hello $name")
No need for complex concatenation

Comments in Kotlin
// Single line comment
/*
 Multi-line
 comment
*/
 
Where Kotlin is Used
Kotlin is used in:
Android Apps 
Backend Development
Web Development
Desktop Applications

Tools Used for Kotlin
To write Kotlin (especially Android apps), we use: Android Studio


Basic Expressions in Kotlin ---

What is an Expression?
In Kotlin, an expression is anything that:
produces a value
This is very important because:
Kotlin treats almost everything as an expression
(not just statements like in Java)
Example:
val sum = 5 + 3
5 + 3 is an expression → result is 8

1. Variables (Expressions storing values)
Variables hold data.
val (Immutable)
val name = "Vineesha"
Cannot be changed later
Safer
Preferred in Kotlin
var (Mutable)
var age = 21
age = 22
Can be changed
Use only when necessary
Key Rule:
Always prefer val over var

2. Data Types
Kotlin automatically detects type (Type Inference).
val num = 10        // Int
val price = 99.9    // Double
val name = "Hi"     // String
val isActive = true // Boolean
Explicit Type (optional)
val num: Int = 10

3. Arithmetic Expressions
Used for calculations.
val a = 10
val b = 5
val sum = a + b
val diff = a - b
val mul = a * b
val div = a / b
val mod = a % b

4. Comparison Expressions
Return Boolean (true/false)
val result = 10 > 5   // true
val check = 5 == 5    // true
Operators:
== equal
!= not equal
> < >= <=

5. Logical Expressions
Used with conditions.
val result = (10 > 5) && (5 > 2)  // true
Operators:
&& → AND
|| → OR
! → NOT

6. Assignment Expressions
Assign values to variables.
var x = 10
x += 5   // x = x + 5 → 15
x -= 2   // 13

7. String Expressions
Kotlin makes strings powerful.
String Templates
val name = "Vineesha"
val age = 21
println("My name is $name and I am $age")
Expression inside String
val a = 5
val b = 3
println("Sum is ${a + b}")
${} allows expressions inside strings

8. if as an Expression (Important)
In Kotlin, if returns a value.
val num = 10
val result = if (num > 0) "Positive" else "Negative"
This is powerful and widely used in Android.

9. Unit Expression
When a function returns nothing:
fun greet() {
    println("Hello")
}
Return type is Unit (like void in Java)

10. Block Expressions
Multiple lines inside {} return last value.
val result = {
    val a = 10
    val b = 20
    a + b
}

Important Concept
In Kotlin:
✔ Expressions return values
✔ You can assign results directly
✔ Code becomes shorter and cleaner
